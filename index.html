Guide d'Administration

Documentation Technique

Fonctionnement Global

  • Moteur R : L'application utilise WebR (Posit). Le code s'exécute directement dans le navigateur de l'étudiant. Pas besoin de serveur R.
  • Base de données : Supabase gère tout le stockage. Les tables principales sont exercices, allowed_students et results.
  • Accès Admin : Le code secret est apex2026R.

Personnalisation du Test (Code source)

🕒 Chrono (Timer) : Modifiez la valeur de EXAM_DURATION_MIN tout au début du fichier app.js (actuellement : 30 min). L'alerte se déclenche automatiquement à 5 min de la fin.

⭐ Barème (Points) : Par défaut, chaque étape vaut 1 point. Pour changer cela, ajoutez simplement @point=2 (ou @pts=5) dans le commentaire de votre solution dans l'onglet de modification. L'application détectera automatiquement la valeur.

✅ Seuil de Validation : La barrette devient verte et le certificat "Validé" apparaît si l'étudiant dépasse 50% (réglable dans updateStudentProgress).

Conseils de Création

Multi-étapes : Pour créer un exercice à plusieurs questions, utilisez le format # Étape 1, # Étape 2 etc. au début de chaque bloc de code.

Gestion des Étudiants : Pour autoriser de nouveaux étudiants, ajoutez leurs emails dans la table allowed_students via l'interface Supabase.

Aperçu : Utilisez toujours le bouton Aperçu avant de sauvegarder pour vérifier le rendu côté étudiant.
